Клятий GiT і його чортинята-команди

  • git config --global user.name blabla - введення імя користувача
  • git config --global user.email blabal@bla - введення електронної пошти користувача
  • git init - git init - створити локальний репозиторій
  • git status - перевірка статусу
  • git add - збереження змін
  • git commit -фіксація змін та запис їх до репозиторія
  • git log - показ усіх коммітів
  • git checkout - переключення гілок
  • git branch - управління гілками
  • git push - відправлення всіх коммітів на віддалений репозиторій
  • git pull - отримання останніх змін віддаленого репозиторію
  • git merge - об'єднання гілок
  • git clone -копіювання репозиторію з віддаленого сервера на локальний комп'ютер

чотиняка який стереже казан від інших

  • .gitignore - файл, який визначає, які файли та папки слід ігнорувати у заданому вихідному коді

ЧОРТИ вищої раси

  • git revert - Команда скасовує дії минулих коммітів , створюючи новий, що містить всі скасовані зміни
  • git reset - скасування змін
  • git rebase - один із двох способів поєднати зміни
  • git cherry-pick - зміни, що вносяться коммітом, і намагається повторно застосувати їх у вигляді нового комміту в поточній гілці
  • git stash - зберігає непідтверджені зміни в окремому сховищі, щоб ви могли повернутися до них пізніше
